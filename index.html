Q2: Print output: 

        OUTPUT:
            1
            0

        REASON:
        The code defines a global variable count with an initial value of 0.
        Inside an IIFE, there's a local variable count declared with the let keyword,
        shadowing the global count. When count is logged within the if block, it displays 1.
        Outside the if block, the local variable goes out of scope, and the global count remains 0, so it logs 0.

Q7: Print the Output: 

            OUTPUT:
                undefined

            REASON:
            The output is undefined because of variable hoisting in JavaScript.
            The variable a is declared inside the function with var,
            so its declaration is hoisted to the top of the function scope,
            making it available but uninitialized at the time of the console.log(a) statement.

Q8: Print the Output: 

            OUTPUT:
                outerArg = 123
                innerArg = 456
                outerVar = a
                innerVar = b
                globalVar = xyz

            REASON:
            outerFunc is invoked with outerArg equal to 123.
            Inside outerFunc, innerFunc is invoked with innerArg equal to 456.
